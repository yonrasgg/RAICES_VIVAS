## Descripción

Como **admin comunitario**, necesito configurar roles y permisos por comunidad para controlar quién accede a qué información, garantizando la soberanía de datos según los principios CARE.

## Criterios de Aceptación

- [ ] El admin comunitario puede crear/editar roles con permisos granulares
- [ ] Los permisos se aplican por módulo y por nivel de acceso CARE
- [ ] Cada comunidad puede tener su propia configuración de roles
- [ ] Los cambios de permisos generan log de auditoría
- [ ] El sistema impide acceso a datos restringidos/ceremoniales sin autorización
- [ ] La configuración funciona offline y se sincroniza
