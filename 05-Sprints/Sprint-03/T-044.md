# ✅ T-044: Implementar sincronización PouchDB ↔ CouchDB

## Objetivo

Desarrollar el módulo de sync bidireccional entre PouchDB (cliente offline) y CouchDB (servidor), cumpliendo los criterios de aceptación de [[US-TRANS-01]]: replicación oportunista, cola de pendientes y resolución de conflictos.

## Criterio de la Rúbrica

> **RF-TRANS-01 — Sincronización offline/online**
> El sistema debe sincronizar datos automáticamente cuando detecta conectividad, manteniendo integridad de datos y resolviendo conflictos.

## Checklist

- [x] Configurar bases de datos PouchDB por módulo (edu, sab, sal)
- [x] Implementar replicación bidireccional con `PouchDB.sync()`
- [x] Agregar detección de estado de conexión (`navigator.onLine` + heartbeat)
- [x] Implementar cola de operaciones pendientes
- [x] Estrategia de conflictos: último timestamp gana con log de versiones
- [x] Hook `useSync()` para consumir estado desde componentes React
