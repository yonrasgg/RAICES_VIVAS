# ✅ T-051: Pruebas de sincronización offline/online

## Objetivo

Validar que el módulo de sincronización funciona correctamente en escenarios de conectividad intermitente, que es el caso de uso principal en territorios indígenas.

## Casos de Prueba

### CP-01: Creación offline + sincronización

1. Desconectar red (modo avión / DevTools offline)
2. Crear un material educativo nuevo
3. Verificar que se guarda en PouchDB local
4. Reconectar red
5. **Esperado:** Material aparece sincronizado en CouchDB ✅

### CP-02: Edición offline + conflicto

1. Editar un registro desde dispositivo A offline
2. Editar el mismo registro desde dispositivo B online
3. Reconectar dispositivo A
4. **Esperado:** Conflicto resuelto por timestamp, versión perdedora en log ✅

### CP-03: Indicador de estado

1. Con red activa: indicador verde "Sincronizado"
2. Desconectar: indicador gris "Sin conexión"
3. Reconectar: indicador azul pulsante "Sincronizando..." → verde ✅
