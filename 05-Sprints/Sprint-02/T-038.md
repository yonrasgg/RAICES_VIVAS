# ✅ T-038: Crear diagrama UML de casos de uso

## Objetivo

Crear el diagrama UML de casos de uso completo del sistema, con agrupación por módulo.

## Criterio de la Rúbrica

> **Criterio 6 — Diagrama de casos de uso UML (4 pts)**
> Presenta un diagrama UML completo, limpio y bien estructurado, con relaciones claras, agrupación lógica y coherencia con la documentación textual.

## Entregable

- [x] Diagrama Mermaid con 5+ actores primarios a la izquierda
- [x] 23 casos de uso agrupados por módulo (subgraphs)
- [x] Relaciones `<<include>>` y `<<extend>>` donde apliquen
- [x] Actores secundarios a la derecha
- [x] Sistemas externos identificados
- [x] Coherencia con la lista de CU y documentación expandida
