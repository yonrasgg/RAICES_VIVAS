## Descripción

Diseñar el modelo entidad-relación del módulo de saberes ancestrales basándose en RF-SAB-01 a RF-SAB-05.

## Criterios de Aceptación (DoD)

- [ ] Entidades identificadas y atributos definidos
- [ ] Niveles de acceso modelados por rol comunitario
- [ ] Jerarquía de categorías (self-reference) representada
- [ ] Diagrama ER en Mermaid renderizable
- [ ] Normalización a 3NF verificada
- [ ] Archivo generado en `04-Arquitectura/Diagramas/ER-SAB.md`

## Plan de Ejecución

> ⚠️ **PLAN ONLY** — Documenta qué se hará, no ejecuta.

### Entidades y Atributos

| Entidad | Atributos (PK en negrita) |
|---------|---------------------------|
| SABER | **id**, titulo, descripcion, tipo, categoria_id (FK), pueblo_id (FK), nivel_acceso, fecha_registro, portador |
| CATEGORIA_SABER | **id**, nombre, descripcion, parent_id (FK self-ref) |
| CONSENTIMIENTO | **id**, saber_id (FK), comunidad_id (FK), tipo, nivel_acceso, otorgante, lengua_explicacion, fecha, activo, evidencia_path |
| ROL_COMUNITARIO | **id**, nombre_rol, nivel_acceso_max, pueblo, territorio |
| PUEBLO | **id**, nombre, lengua, territorio_principal, poblacion_aprox |
| MULTIMEDIA_SABER | **id**, saber_id (FK), tipo_media, ruta, tamano_mb, formato |

### Relaciones

| Relación | Cardinalidad | Nota |
|----------|--------------|------|
| SABER → CATEGORIA_SABER | N — 1 | Clasificación temática |
| SABER → PUEBLO | N — 1 | Origen cultural |
| SABER → CONSENTIMIENTO | 1 — N | Puede tener múltiples consentimientos |
| SABER → MULTIMEDIA_SABER | 1 — N | Audio, video, imagen |
| CATEGORIA_SABER → CATEGORIA_SABER | self-ref | Jerarquía padre-hijo |
| ROL_COMUNITARIO | — | Gobierna niveles de acceso a SABER |

### Especificaciones Técnicas

- **Formato**: Mermaid `erDiagram`
- **Normalización**: 3NF
- **Convención**: `snake_case` para tablas y columnas
- **Archivo destino**: `04-Arquitectura/Diagramas/ER-SAB.md`
- **Requerimientos cubiertos**: RF-SAB-01 a RF-SAB-05
- **Decisión arquitectónica**: ADR-009 (modelo de consentimiento informado)

## Notas

- El campo `nivel_acceso` en SABER y CONSENTIMIENTO es clave para la gobernanza cultural — revisar ADR-009.
- La self-reference en CATEGORIA_SABER permite taxonomías jerárquicas (ej. Medicina → Plantas → Preparaciones).
- `evidencia_path` en CONSENTIMIENTO almacena ruta a documento escaneado o audio de consentimiento oral.
