# T-029 · Preparar instrumentos de validación

> Tarea pendiente — Sprint-02 / Avance 2.

## Descripción

Elaborar los instrumentos para validar los prototipos con usuarios finales:
- Guía de prueba de usabilidad (tareas + métricas)
- Formulario de retroalimentación para comunidades
- Rúbrica de evaluación heurística
- Checklist de accesibilidad (WCAG 2.1 AA)

## Criterios de Aceptación

- [ ] Guía de entrevista semiestructurada con 15-20 preguntas
- [ ] Secciones cubren UX, funcionalidad y pertinencia cultural
- [ ] Formato checklist en Markdown
- [ ] Validada por el equipo antes de uso
- [ ] Documentada en `02-Investigación/Encuestas/`

## Plan de Ejecución

> ⚠️ **PLAN ONLY** — Documenta qué se hará, no ejecuta.

### Instrumento: Guía de Entrevista Semiestructurada (Walkthrough)

| Sección | Enfoque | Preguntas aprox. |
|---------|---------|------------------|
| 1. Presentación del módulo | Contexto y propósito general del módulo evaluado | 2-3 |
| 2. Flujo de usuario | Navegación, pasos del flujo principal, lógica de pantallas | 3-4 |
| 3. Usabilidad | Claridad visual, facilidad de uso, feedback del sistema | 4-5 |
| 4. Completitud funcional | ¿Cubre los RF? ¿Faltan campos o acciones? | 3-4 |
| 5. Pertinencia cultural | Representación apropiada, lenguaje inclusivo, respeto a cosmovisión | 3-4 |

### Especificaciones

- **Formato**: Markdown checklist (`- [ ]` por pregunta)
- **Directorio destino**: `02-Investigación/Encuestas/`
- **Método**: Internal walkthrough — 3 miembros del equipo revisan módulos cruzados
- **Total preguntas**: 15-20
- **Cobertura**: UX, funcionalidad, pertinencia cultural

## Notas

- Este instrumento se usará en T-030 (planificación de entrevistas).
- Al ser fase académica, se aplica como walkthrough interno, no con comunidades.
