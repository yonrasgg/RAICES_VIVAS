## Descripción

Diseñar el modelo entidad-relación del módulo educativo basándose en los RF-EDU-01 a RF-EDU-06.

## Criterios de Aceptación (DoD)

- [ ] Entidades identificadas y atributos definidos
- [ ] Relaciones y cardinalidades documentadas
- [ ] Diagrama ER en Mermaid renderizable
- [ ] Normalización a 3NF verificada
- [ ] Archivo generado en `04-Arquitectura/Diagramas/ER-EDU.md`

## Plan de Ejecución

> ⚠️ **PLAN ONLY** — Documenta qué se hará, no ejecuta.

### Entidades y Atributos

| Entidad | Atributos (PK en negrita) |
|---------|---------------------------|
| DOCENTE | **id**, nombre, cedula, lengua_materna, territorio_id, especialidad, fecha_registro |
| MATERIAL_EDUCATIVO | **id**, titulo, tipo, lengua, nivel, docente_id (FK), fecha_creacion, ruta_archivo |
| SESION_PRACTICA | **id**, material_id (FK), estudiante_id (FK), fecha, puntuacion, completada |
| EVALUACION | **id**, sesion_id (FK), tipo, resultado, retroalimentacion |
| ESTUDIANTE | **id**, nombre, nivel, lengua_materna, comunidad_id (FK) |
| COMUNIDAD | **id**, nombre, territorio, pueblo, adi_nombre |

### Relaciones

| Relación | Cardinalidad |
|----------|--------------|
| DOCENTE → MATERIAL_EDUCATIVO | 1 — N |
| MATERIAL_EDUCATIVO → SESION_PRACTICA | 1 — N |
| ESTUDIANTE → SESION_PRACTICA | 1 — N |
| SESION_PRACTICA → EVALUACION | 1 — 1 |
| COMUNIDAD → DOCENTE | 1 — N |
| COMUNIDAD → ESTUDIANTE | 1 — N |

### Especificaciones Técnicas

- **Formato**: Mermaid `erDiagram`
- **Normalización**: 3NF
- **Convención**: `snake_case` para tablas y columnas
- **Archivo destino**: `04-Arquitectura/Diagramas/ER-EDU.md`
- **Requerimientos cubiertos**: RF-EDU-01 a RF-EDU-06
