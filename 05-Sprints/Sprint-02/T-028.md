# T-028 · Elaborar wireframes módulo SAL

> Tarea pendiente — Sprint-02 / Avance 2.

## Descripción

Diseñar wireframes de baja/media fidelidad para las pantallas del módulo de Salud Comunitaria:
- Dashboard de indicadores de salud
- Registro de atención comunitaria
- Calendario de brigadas médicas
- Ficha de paciente / seguimiento
- Reportes por comunidad

## Criterios de Aceptación

- [ ] Wireframes cubren los flujos principales del RF-SAL-01
- [ ] Mínimo 3 pantallas diseñadas
- [ ] Layout mobile-first (Android tablet)
- [ ] Placeholders de i18n en todos los textos
- [ ] Consideran contexto de salud rural indígena
- [ ] Revisados por el equipo en sesión de validación
- [ ] Exportados como PNG/SVG en `04-Arquitectura/Prototipos/WF-SAL/`

## Plan de Ejecución

> ⚠️ **PLAN ONLY** — Documenta qué se hará, no ejecuta.

### Pantallas a Diseñar

| # | Pantalla | Contenido clave |
|---|----------|-----------------|
| 1 | **Registro de Paciente** | Formulario con ID único temporal, datos personales, selección de comunidad, lengua materna |
| 2 | **Historial Médico** | Timeline de consultas, filtros por tipo de consulta, campo de notas culturales visible |
| 3 | **Calendario de Citas** | Vista semanal/mensual, programación de brigadas de salud, alertas de seguimiento |

### Especificaciones de Diseño

- **Herramienta**: Excalidraw (plugin Obsidian) o Figma
- **Directorio destino**: `04-Arquitectura/Prototipos/WF-SAL/`
- **Layout**: Mobile-first, optimizado para Android tablet
- **Internacionalización**: Placeholders `{{i18n.key}}` en todos los textos visibles

## Notas

- Depende de T-024 (modelo ER SAL) para conocer las entidades del módulo de salud.
- El impedimento RV-45 (datos médicos necesarios) afecta directamente el formulario de Registro de Paciente.
- El campo de ID temporal es crítico para pacientes sin cédula en comunidades remotas.
