## Descripción

Diseñar el modelo entidad-relación del módulo de salud comunitaria basándose en RF-SAL-01 a RF-SAL-05.

## Criterios de Aceptación (DoD)

- [x] Entidades identificadas y atributos definidos
- [x] Confidencialidad por roles modelada
- [x] Relaciones y cardinalidades documentadas
- [x] Diagrama ER en Mermaid renderizable
- [x] Normalización a 3NF verificada
- [ ] Archivo generado en `04-Arquitectura/Diagramas/ER-SAL.md`

## Plan de Ejecución

> ⚠️ **PLAN ONLY** — Documenta qué se hará, no ejecuta.

### Entidades y Atributos

| Entidad | Atributos (PK en negrita) |
|---------|---------------------------|
| PACIENTE | **id**, nombre, cedula_o_temporal, fecha_nacimiento, sexo, comunidad_id (FK), lengua_materna, tipo_sangre |
| HISTORIAL_MEDICO | **id**, paciente_id (FK), fecha, tipo_consulta, diagnostico, tratamiento, profesional, notas_culturales |
| CITA | **id**, paciente_id (FK), fecha_programada, ebais_o_comunidad, tipo, estado, profesional |
| CAMPANA_SALUD | **id**, titulo, tipo, fecha_inicio, fecha_fin, territorio_id (FK), responsable |
| ALERTA | **id**, paciente_id (FK), tipo_alerta, fecha_generada, urgencia, leida, accion_tomada |
| EBAIS | **id**, nombre, territorio_id (FK), distancia_km, personal_count |

### Relaciones

| Relación | Cardinalidad | Nota |
|----------|--------------|------|
| PACIENTE → HISTORIAL_MEDICO | 1 — N | Registro longitudinal |
| PACIENTE → CITA | 1 — N | Citas programadas y completadas |
| PACIENTE → ALERTA | 1 — N | Alertas de seguimiento |
| CAMPANA_SALUD → territorio | 1 — N | Cobertura territorial |
| EBAIS → COMUNIDAD | serves | Relación de servicio |

### Especificaciones Técnicas

- **Formato**: Mermaid `erDiagram`
- **Normalización**: 3NF
- **Convención**: `snake_case` para tablas y columnas
- **Archivo destino**: `04-Arquitectura/Diagramas/ER-SAL.md`
- **Requerimientos cubiertos**: RF-SAL-01 a RF-SAL-05

## Notas

- `cedula_o_temporal` permite identificar pacientes sin cédula mediante ID temporal — necesario para comunidades remotas.
- `notas_culturales` en HISTORIAL_MEDICO captura contexto de medicina tradicional relevante al diagnóstico.
- El impedimento RV-45 (datos médicos necesarios) debe resolverse antes de finalizar atributos de PACIENTE e HISTORIAL.
