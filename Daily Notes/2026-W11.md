## 📊 Métricas de la Semana

```dataviewjs
const ws = dv.current().week_start;
const we = dv.current().week_end;
const tasks = dv.pages('"05-Sprints"').where(t => t.type === "task" || t.type === "subtask");

const doneThisWeek = tasks.where(t =>
  t.status === "done" && t.completed &&
  dv.date(t.completed) >= dv.date(ws) &&
  dv.date(t.completed) <= dv.date(we)
).length;

const inProgress = tasks.where(t => t.status === "in-progress" || t.status === "review").length;
const blocked = tasks.where(t => t.status === "blocked").length;

const parseH = (v) => { if (!v) return 0; if (typeof v === "number") return v; const m = String(v).match(/\d+/); return m ? parseInt(m[0]) : 0; };
let hoursWeek = 0;
for (const t of tasks.where(t => t.status === "done" && t.completed && dv.date(t.completed) >= dv.date(ws) && dv.date(t.completed) <= dv.date(we))) {
  hoursWeek += t.effort_actual ? parseH(t.effort_actual) : parseH(t.effort);
}

const adrs = dv.pages('"01-Proyecto/Decisiones"').where(d => d.type === "adr" && d.date && dv.date(d.date) >= dv.date(ws) && dv.date(d.date) <= dv.date(we)).length;
const meetings = dv.pages('"07-Reuniones"').where(m => m.type === "meeting" && m.date && dv.date(m.date) >= dv.date(ws) && dv.date(m.date) <= dv.date(we)).length;

const totalTasks = tasks.length;
const totalDone = tasks.where(t => t.status === "done").length;
const pct = totalTasks > 0 ? Math.round((totalDone / totalTasks) * 100) : 0;

dv.table(
  ["📊 Métrica", "Valor"],
  [
    ["✅ Completadas esta semana", doneThisWeek],
    ["⏱️ Horas ejecutadas", `${hoursWeek}h`],
    ["🔄 En progreso", inProgress],
    ["🚫 Bloqueadas", blocked],
    ["🏗️ ADR esta semana", adrs],
    ["📝 Reuniones", meetings],
    ["📈 Progreso global", `${pct}% (${totalDone}/${totalTasks})`]
  ]
);
```

## 📝 Notas y Decisiones

- **ADR-008 aprobado (Stack Tecnológico):** Se formalizó la decisión de PWA con React + PouchDB + CouchDB + i18next. El análisis FODA ponderado mostró ventaja sobre Flutter y Vue para el contexto offline-first. Debate sobre si React Native sería mejor se descartó por mayor mantenimiento.
- **ADR-009 aprobado (Gobernanza Cultural):** Elkin presentó los principios CARE (Collective benefit, Authority to control, Responsibility, Ethics) aplicados al manejo de saberes ancestrales. Se definieron 4 niveles de acceso: público, comunidad, restringido-cultural, sagrado. Se requiere consentimiento comunitario para nivel restringido y sagrado.
- **Modelos ER:** Se avanzó en la definición de entidades para los módulos:
  - EDU: 6 entidades principales (Recurso, Lengua, CategoríaEducativa, Actividad, ProgresoEstudiante, MediaContent)
  - SAB: 7 entidades con niveles CARE integrados (SaberAncestral, Comunidad, Territorio, NivelAcceso, Custodio, Validación, MediaSaber)
  - SAL: 6 entidades (BrigadaSalud, CentroSalud, IndicadorComunitario, AlertaSanitaria, PersonalSalud, RegistroVisita)
- **Investigación de conectividad:** Geovanny compiló datos de SUTEL y FONATEL sobre cobertura 3G/4G en territorios indígenas. Resultado: 17 de 24 territorios tienen cobertura parcial, 7 sin cobertura estable → confirma necesidad de arquitectura offline-first con sync oportunista.
- Santiago comenzó a delinear los flujos de las pantallas SAL para preparar wireframes
- 📋 **Reunión formal:** [[07-Reuniones/MIN-004|MIN-004 — Aprobación de ADRs y Avance de Modelos ER]] (2026-03-14)
