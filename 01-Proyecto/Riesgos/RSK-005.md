# RSK-005: Complejidad del frontmatter si se crean notas manualmente

## Descripción

El frontmatter YAML del proyecto tiene entre 10 y 20 campos por tipo de nota ([[ADR-005]]). Si un integrante crea una nota manualmente (sin usar QuickAdd), es muy probable que omita campos obligatorios, use valores incorrectos, o rompa la indentación YAML.

## Causa Raíz

- YAML es sensible a indentación y formato
- Cada tipo de nota tiene un esquema diferente
- La tentación de "copiar y pegar" de otra nota introduce errores sutiles
- No hay validación automática del frontmatter (solo Linter para formato)

## Impacto Detallado

| Dimensión | Descripción del Impacto |
|-----------|------------------------|
| **Alcance** | Notas con campos faltantes afectan queries SQLSeal |
| **Calendario** | Debugging de queries que fallan por datos faltantes |
| **Calidad** | RTM, Dashboard y Métricas muestran datos incompletos |

## Trigger (Indicador de Alerta)

- SQLSeal queries devuelven datos incompletos o incorrectos
- Notas sin campo `type:` o `id:` en el frontmatter
- Linter reporta errores de YAML

## Plan de Respuesta

### Acciones Preventivas (Antes)

- [ ] Regla de equipo: NUNCA crear notas en blanco — siempre usar QuickAdd (`Ctrl+P → "Nueva..."`) 📅 2026-03-05
