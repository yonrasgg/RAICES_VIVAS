# RSK-014: Sin validación con usuarios reales en fase de diseño (Sprint-02)

## Control Rápido

| Campo | Valor |

| **Estado** | `INPUT[suggester(option(open), option(mitigated), option(closed), option(accepted)):status]` |
| **Probabilidad** | `INPUT[suggester(option(alta), option(media), option(baja)):probability]` |

| **Estrategia** | `INPUT[suggester(option(mitigar), option(transferir), option(aceptar), option(evitar)):strategy]` |

## Descripción

Los artefactos de diseño del Sprint-02 (modelos ER, wireframes, decisiones de stack y gobernanza) no se validan con usuarios finales reales (docentes indígenas, custodios culturales, auxiliares de salud). La validación se limita a un walkthrough interno entre los 3 integrantes del equipo.

Se acepta como limitación documentada del proyecto académico.

> Identificado en [[MIN-003]].

## Causa Raíz

- Coordinación con CONAI no se concretó a tiempo ([[RSK-013]])
- Logística de acceso a territorios indígenas
- Tiempo limitado del cuatrimestre para fase de diseño

## Impacto Detallado

| Dimensión | Impacto |
|-----------|---------|
| **Alcance** | Los diseños podrían no reflejar necesidades reales |
| **Calendario** | Se descubre tarde si hay gaps (Sprint-03/04) |
| **Calidad** | Sesgo del equipo en la evaluación de sus propios artefactos |
| **Recursos** | Retrabajo potencial en sprints posteriores |

## Trigger

Artefactos de diseño se entregan en Avance 2 sin feedback de usuarios reales.

## Plan de Respuesta

### Acciones Preventivas

- [x] Walkthrough interno robusto con guía semi-estructurada ([[ADR-013]])
- [x] Instrumentos de validación diseñados para reuso con usuarios reales ([[ADR-016]])
- [x] Documentado como limitación explícita en Avance 2

### Contingencia

- La validación con usuarios reales se pospone a Sprint-03/04
- Diseños son lo suficientemente flexibles para adaptarse (CouchDB schemaless)
- Instrumentos SUS y WCAG están listos para aplicarse cuando haya acceso
