# RSK-004: Migración futura a GitHub Projects podría causar inconsistencias

## Descripción

La decisión de adoptar GitHub Projects en la fase de implementación ([[ADR-003]]) introduce el riesgo de mantener dos fuentes de información (Obsidian + GitHub Projects). Si no se define claramente qué vive dónde, el equipo podría actualizar un sistema y olvidar el otro.

## Causa Raíz

- Se usan dos herramientas para gestión del proyecto
- No existe sincronización automática entre Obsidian y GitHub Projects
- Diferentes integrantes podrían preferir una herramienta sobre otra

## Impacto Detallado

| Dimensión | Descripción del Impacto |
|-----------|------------------------|
| **Alcance** | Información contradictoria entre sistemas |
| **Calendario** | Tiempo extra para reconciliar datos |
| **Calidad** | Métricas y dashboards podrían ser incorrectos |

## Trigger (Indicador de Alerta)

- Datos duplicados o contradictorios entre Obsidian y GitHub Projects
- Un integrante actualiza solo en GitHub Projects sin reflejar en Obsidian
- Dashboard de Obsidian muestra métricas diferentes a GitHub Projects

## Plan de Respuesta

### Acciones Preventivas (Antes)

- [ ] Definir claramente la división de responsabilidades: Obsidian = documentación + trazabilidad, GitHub Projects = ejecución de código 📅 2026-04-01
- [ ] Documentar el protocolo de sincronización manual entre ambos sistemas 📅 2026-04-01

### Acciones de Contingencia (Si ocurre)

| # | Acción | Responsable | Recursos necesarios |
|---|--------|------------|---------------------|
| 1 | Obsidian es siempre la fuente de verdad | Equipo | Convención de equipo |
