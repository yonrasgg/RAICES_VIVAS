# RSK-002: Conflictos de sincronización Git entre 3 integrantes

## Descripción

Cuando 2 o más integrantes editan el mismo archivo simultáneamente, Git genera conflictos de merge. En archivos Markdown con frontmatter YAML, un conflicto mal resuelto puede corromper el frontmatter y romper queries Dataview.

## Causa Raíz

- 3 integrantes trabajan en el mismo vault de forma asíncrona
- Auto-push/pull cada 10 minutos puede no ser suficiente si editan rápido
- Archivos YAML son sensibles a errores de indentación

## Impacto Detallado

| Dimensión | Descripción del Impacto |
|-----------|------------------------|
| **Alcance** | El archivo afectado queda inutilizable hasta resolver el conflicto |
| **Calendario** | 15-30 min para resolver un conflicto manualmente |
| **Calidad** | Frontmatter corrupto causa errores silenciosos en Dataview |

## Trigger (Indicador de Alerta)

- Merge conflicts frecuentes (más de 2 por semana)
- Archivos con marcadores `<<<<<<<` en el vault
- Queries Dataview devuelven resultados inesperados
