# ADR-006: Templates automatizados con Templater + QuickAdd

## Contexto

Dado el esquema de frontmatter estricto ([[ADR-005]]), se necesita un mecanismo que evite la creación manual de notas. El sistema debe auto-generar IDs consecutivos, pre-rellenar campos, y renombrar archivos automáticamente.

> Decisión originada en [[MIN-001]].

## Decisión

**12 macros QuickAdd con templates Templater que:**

1. **Auto-ID** — Calcula el siguiente ID consecutivo via `dv.pages()` + `Math.max()`
2. **Auto-rename** — Renombra el archivo al ID generado (`tp.file.rename()`)
3. **Prompts interactivos** — Solicita datos al usuario con `tp.system.prompt()` y `tp.system.suggester()`
4. **Promoción desde minutas** — Extrae texto seleccionado y detecta la minuta origen
5. **Severidad automática** — En riesgos, calcula `probabilidad × impacto`

| Macro | Template | Auto-ID |
|-------|----------|---------|
| ✅ Nueva Tarea | `_template-tarea.md` | T-XXX |
| 🏗️ Nuevo ADR | `_template-adr.md` | ADR-XXX |
| ⚠️ Nuevo Riesgo | `_template-riesgo.md` | RSK-XXX |
| 📋 Promover Action Item | `_template-tarea-from-minuta.md` | T-XXX |
| 🏗️ Promover Decisión | `_template-adr-from-minuta.md` | ADR-XXX |
| ⚠️ Promover Riesgo | `_template-riesgo-from-minuta.md` | RSK-XXX |

## Consecuencias

### Positivas

- Frontmatter siempre consistente (no hay errores manuales)
- IDs nunca se duplican ni se saltan
- Trazabilidad bidireccional automática (source, requirement)
- Baja la barrera de entrada para Elkin y Santiago

### Negativas / Riesgos

- [[RSK-001]]: Si falla el sistema de templates, solo Geovanny puede repararlo
- [[RSK-005]]: Dependencia total en que el equipo use QuickAdd (mitigado con convención)
