# ADR-002: Git + GitHub para control de versiones y colaboración

## Contexto

El vault de Obsidian necesita un mecanismo de sincronización entre 3 integrantes que permita historial granular, resolución de conflictos y respaldo remoto. Se requiere que funcione de forma asíncrona (no simultánea).

> Decisión originada en [[MIN-001]].

## Opciones Consideradas

| # | Opción | Pros | Contras |
|---|--------|------|---------|
| 1 | **Git + GitHub** | Historial completo, merge granular, gratuito, estándar | Requiere conocimiento de Git |
| 2 | **Obsidian Sync** | Integrado, simple | Costo mensual, sin historial granular |
| 3 | **Dropbox/OneDrive** | Simple | Conflictos de sincronización, sin merge inteligente |

## Decisión

**Opción seleccionada: Git + GitHub (repositorio privado)**

- Plugin obsidian-git con auto commit/push/pull cada 10 minutos
- Repositorio: `github.com/yonrasgg/RAICES_VIVAS`
- Cada integrante trabaja en su copia local y sincroniza automáticamente

## Consecuencias

### Positivas

- Historial completo de cada cambio con timestamps y autor
- Respaldo automático en la nube (GitHub)
- Capacidad de revertir cambios y resolver conflictos
- Gratuito para repositorios privados

### Negativas / Riesgos

- [[RSK-002]]: Conflictos de sincronización si 2 personas editan el mismo archivo
- [[RSK-006]]: Pérdida de trabajo si falla la sincronización
- Requiere que cada integrante configure Git en su máquina
