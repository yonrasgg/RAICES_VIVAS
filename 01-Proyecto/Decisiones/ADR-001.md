## Contexto

El equipo necesitaba una herramienta que sirviera como **punto único de verdad** para toda la documentación, trazabilidad, métricas y gestión del proyecto Raíces Vivas. Los requisitos clave eran:

- Soporte para Markdown puro (portabilidad)
- Consultas dinámicas sobre metadatos (Dataview)
- Templates interactivos con lógica (Templater + QuickAdd)
- Control de versiones integrado (Git)
- Funcionalidad offline
- Bajo costo (open-source)

> Decisión originada en [[MIN-001]].

## Opciones Consideradas

| # | Opción | Pros | Contras |
|---|--------|------|---------|
| 1 | **Obsidian** | Local-first, Markdown puro, extensible con plugins, Git nativo, Dataview para queries | Curva de aprendizaje, no es colaborativo en tiempo real |
| 2 | **Notion** | Colaboración en tiempo real, UI intuitiva | Vendor lock-in, sin Markdown puro, sin offline real, costo |
| 3 | **Google Docs + Sheets** | Colaboración fácil, familiar | Sin trazabilidad automática, sin frontmatter, difícil de versionar |
| 4 | **Jira + Confluence** | Estándar de industria para gestión ágil | Excesivo para proyecto académico, costo, complejidad |

## Decisión

**Opción seleccionada: Obsidian**

Obsidian se adoptó como centro de control único del proyecto porque:
1. **Markdown puro** garantiza portabilidad y longevidad de los datos
2. **Dataview** permite dashboards, RTM y métricas dinámicas sin herramientas externas
3. **Templater + QuickAdd** automatiza la creación de notas con frontmatter consistente
4. **Git** integrado permite versionado y colaboración asíncrona
5. **22 plugins** cubren todas las necesidades del proyecto sin salir de la herramienta

## Consecuencias

### Positivas

- Un solo sistema para documentación, gestión y trazabilidad
- Todo versionado en Git con historial completo
- Consultas en tiempo real sobre el estado del proyecto
- Sin dependencia de servicios en la nube

### Negativas / Riesgos

- [[RSK-001]]: Dependencia en Geovanny para configuración del vault
- [[RSK-003]]: Curva de aprendizaje para Elkin y Santiago
- [[RSK-005]]: Complejidad del frontmatter si se crean notas manualmente
- No hay colaboración simultánea en tiempo real (mitigado con Git auto-sync)

### Compromisos (Trade-offs)

- Se sacrifica la edición simultánea a cambio de control total del formato
- Se requiere inversión inicial en configuración de plugins y templates
