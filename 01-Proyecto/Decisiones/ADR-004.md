# ADR-004: Estructura del vault con carpetas numeradas y convenciones de naming

## Contexto

Se necesita una estructura de directorio que sea intuitiva, escalable y que permita localizar cualquier artefacto del proyecto en menos de 3 clics. La numeración de carpetas garantiza orden visual y agrupamiento lógico.

> Decisión originada en [[MIN-001]].

## Decisión

**Estructura adoptada: Carpetas numeradas 00-09 + 99**

```
00-Dashboard/     ← KPIs y navegación
01-Proyecto/      ← Gobierno (Charter, Scope, Decisiones, Riesgos)
02-Investigación/ ← Fase de descubrimiento
03-Requerimientos/← RF y RNF por módulo
04-Arquitectura/  ← Diseño técnico
05-Sprints/       ← Gestión iterativa (tareas)
06-Entregables/   ← Documentos de entrega
07-Reuniones/     ← Minutas
08-Recursos/      ← PDFs, imágenes, scripts
09-QA/            ← Control de calidad
99-Templates/     ← Plantillas del vault
```

## Consecuencias

### Positivas

- Orden visual inmediato en el file explorer
- Cada tipo de artefacto tiene un lugar definido
- Escalable (se pueden agregar carpetas intermedias)
- Compatible con cualquier sistema operativo

### Negativas / Riesgos

- La numeración puede limitar reorganizaciones futuras
- Nuevos integrantes deben aprender la convención
